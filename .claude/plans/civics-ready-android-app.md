# Plan: Civics Ready — Android App for USCIS Civics Test

## Overview
A native Android app that helps US naturalization applicants study for the 2025 USCIS Civics Test (128 questions). Source PDF: `2025-Civics-Test-128-Questions-and-Answers.pdf` in repo root.

---

## Tech Stack
| Concern | Choice |
|---|---|
| Language | Kotlin |
| UI | Jetpack Compose + Material 3 |
| Architecture | Single-module MVVM + Clean Architecture |
| DI | Hilt |
| Navigation | Navigation Compose |
| Local DB | Room (progress tracking) |
| Preferences | DataStore Preferences (zip code, 65/20 flag) |

## Data Model

### CivicsQuestion
```kotlin
data class CivicsQuestion(
    val id: Int,                           // 1–128
    val section: Section,
    val text: String,
    val acceptableAnswers: List<String>,   // any one (or N) is correct
    val minimumAnswersRequired: Int = 1,   // "Name 5" questions = 5
    val dynamicAnswerType: DynamicAnswerType? = null,
    val isFor6520: Boolean = false         // asterisked in PDF
)
```

### DynamicAnswerType
Questions whose answers depend on user location or current officeholders:
- `GOVERNOR` → Q61
- `SENATOR` → Q23 (accepts either senator1 or senator2)
- `REPRESENTATIVE` → Q29
- `STATE_CAPITAL` → Q62
- `PRESIDENT` → Q38 (hardcoded: Donald Trump)
- `VICE_PRESIDENT` → Q39 (hardcoded: JD Vance)
- `SPEAKER_OF_HOUSE` → Q30 (hardcoded: Mike Johnson)
- `CHIEF_JUSTICE` → Q57 (hardcoded: John Roberts)

### 65/20 Asterisked Question IDs
`2, 7, 12, 20, 30, 36, 38, 39, 44, 52, 61, 66, 74, 78, 86, 94, 113, 115, 121, 126`

### Questions requiring multiple answers (minimumAnswersRequired > 1)
| Q# | Required | Topic |
|---|---|---|
| 10 | 2 | Ideas from Declaration + Constitution |
| 48 | 2 | Cabinet-level positions |
| 65 | 3 | Rights of everyone in the US |
| 67 | 2 | Oath of Allegiance promises |
| 69 | 2 | Examples of civic participation |
| 81 | 5 | Original 13 states |
| 126 | 3 | National US holidays |

## Civic Officials Data Files

### officials.json
Maps state code → `{name, governor, senator1, senator2, capital}` for all 50 states + DC.
Data as of January 2025.

### districts.json
Maps `"ST-N"` → representative name for all 435 House districts (119th Congress).
Special cases: DC uses district 0 (no voting rep), at-large states use `-1`.

### zip_codes.json
Maps zip code → `{s: stateCode, d: districtNumber}`.
349 entries covering all 50 states + DC (major cities and state capitals).
Generated by `c:\src\uscis-civics-test\gen_zip_codes.py`.

**Expanding zip coverage:** Run `gen_zip_codes.py` with additional entries or replace with
data from the US Census Bureau ZCTA-to-congressional-district crosswalk file.

## Screens & Navigation

```
home → practice         (PracticeScreen)
home → test             (TestScreen → navigates to result on completion)
     → result           (TestResultScreen — shares TestViewModel via nav backstack)
home → settings         (SettingsScreen)
```

### Home Screen
- Stats row (questions practiced, accuracy, mode)
- Location card showing resolved officials (tappable → Settings)
- Practice Mode button (all 128 or 20 for 65/20)
- Test Mode button (20 or 10 for 65/20)

### Practice Mode
- Shuffle questions → FlashCard → ✓/✗ → next
- Progress bar (green) across top
- Progress persisted to Room after each answer
- Summary screen on finish or early exit

### Test Mode
- 20 random questions (10 for 65/20) — same FlashCard UX
- Red progress bar to differentiate from practice
- Navigates to TestResultScreen when all 20 answered

### TestResultScreen
- PASS / FAIL banner (green / red)
- Score: X / 20 (or X / 10)
- Per-question list: ✓ correct, ✗ incorrect with correct answer shown
- Try Again + Back to Home

### Settings Screen
- Zip code field + Look Up button → resolves officials offline
- Shows state, capital, governor, senator 1, senator 2, representative
- Federal officials section (hardcoded + last-updated note)
- 65/20 mode toggle with explanation

### FlashCard Component
- Animated Y-axis flip (`animateFloatAsState`, 400ms)
- Front: question text, section label, "Tap to reveal"
- Back: acceptable answers list, "Got it" / "Missed" buttons
- Swipe gesture: right = correct, left = incorrect (threshold 120px)
